# Инструкция для Claude Code

## Кто я
Виталий Семьянов. Преподаватель бачаты. Продюсер онлайн-запусков.
Директор по маркетингу танцевальной школы D'Arky Dance в Новосибирске.

## Как работать с этим репозиторием
Это система управления проектами. Каждая папка — отдельный проект.

## Мои проекты
1. **Фитнес-запуск** (`/fitness-online`) — онлайн-курс по похудению с тренером Ирина Денисова
2. **Курс по футворкам** (`/bachata-course_footworks`) — авторский курс для танцоров бачаты
3. **D'Arky Dance** (`/darky-dance`) — танцевальная школа бачаты в Новосибирске (Фрунзе 80)
4. **Блог** (`/bachata-blog`) — экспертный контент о танцах

## Правила работы

### Перед выполнением любой задачи:
1. Прочитай этот файл (CLAUDE.md)
2. Прочитай `brief.md` нужного проекта — пойми контекст
3. Прочитай `decisions.md` — не противоречь принятым решениям
4. Проверь `log.md` — пойми, на каком мы этапе

### После выполнения задачи:
1. Обновляй `log.md` проекта с датой и кратким описанием того, что было сделано
2. Если нужно моё решение — отметь в `log.md` как **[РЕШЕНИЕ ТРЕБУЕТСЯ]**

### При работе с файлами:
1. Редактируй существующие файлы — не создавай дубликаты
2. Не удаляй существующий контент, только дополняй
3. Сохраняй результат в соответствующий `.md` файл проекта

### Когда пишешь пост или любой SMM-текст для D'Arky Dance:
1. Прочитай `darky-dance/smm/copywriting-rules.md` — там правила стиля, структура по типам постов и стоп-лист
2. Прочитай `darky-dance/brand-voice.md` секцию «Голос в соцсетях» — фирменные маркеры, наши слова, типы хуков, правила CTA
3. При необходимости — загляни в `darky-dance/smm/reference-texts.md` (реальные опубликованные тексты) и `darky-dance/smm/style-analysis.md` (детальный разбор паттернов)
4. Используй шаблон из `darky-dance/smm/templates/` нужного типа поста

### Чего не делать:
- Не выдумывай факты обо мне или моих проектах
- Не меняй `decisions.md` без моей явной команды
- Не меняй `brief.md` без моей явной команды
- Не додумывай — если нужна информация от меня, спроси напрямую

### Если видишь противоречие между файлами:
- Останови работу и сообщи мне, какие именно файлы и в чём противоречат
- Не принимай решение сам, какой файл «правильный»

## Формат
- Все файлы — markdown
- Язык — русский
- Стиль — конкретный, без воды, с примерами
- Если не хватает информации — скажи, что именно нужно узнать

## Мой стиль
- Простой язык, без корпоративного буллшита
- Конкретные действия, а не абстрактные советы
- Если нужна информация от меня — спроси, не додумывай
