# Структура лендинга — запуск «Плоский живот»

## Для чего этот документ
Этот файл задает блоки лендинга и логику подачи, чтобы страница вела посетителя к оплате без провалов по смыслу.

## Цель страницы
- Главное действие пользователя: вступить в чат предзаписи на `/` и оплатить доступ на `/pay`.
- Вторичное действие: оставить телефон как резервный контакт и перейти в поддержку при вопросах по оплате.
- Формат страницы: 2 отдельные страницы (`/` и `/pay`) + 2 технические страницы (`/prechat-instruction`, `/pay/success`).

## Логика переходов (обязательная)
- Реклама -> `умныйживот.рф` -> отправка формы -> `/prechat-instruction`.
- `/prechat-instruction` -> кнопка вступления в чат предзаписи (Telegram).
- Из чата предзаписи -> переход на `умныйживот.рф/pay`.
- `умныйживот.рф/pay` -> YooKassa -> `/pay/success` (при успехе).
- `/pay/success` -> кнопка вступить в закрытый чат.
- При отмене/ошибке оплаты возвращаем на `/pay?payment=failed`.

## Формы и поля
### Форма предзаписи (`POST /api/prechat-lead`)
- `name` (обязательное, 2-50 символов).
- `phone` (необязательное, строка, валидация по формату РФ).
- `consent_privacy` (обязательный checkbox).
- Служебные поля: `lead_id`, `landing_page`, `ts_client`, `utm_*`, `first_touch_*`, `last_touch_*`.
- Ответ API: `ok`, `lead_id`, `redirect_url` (`/prechat-instruction`).

### Форма/кнопка оплаты (`POST /api/payment/create`)
- Вход: `lead_id`, `name`, `phone?`, `email?`, `utm_*`.
- Выход: `ok`, `payment_id`, `amount`, `price_stage`, `confirmation_url`.
- После ответа фронт делает redirect на `confirmation_url` YooKassa.

## API-контракты (минимум)
### `POST /api/prechat-lead`
Назначение: сохранить лид в Google Sheets (`leads`) и отправить Telegram-уведомление.

### `POST /api/payment/create`
Назначение: рассчитать цену по МСК, создать платеж YooKassa с metadata, записать `pending` в `payments`.

### `POST /api/payment/webhook`
Назначение: обновить статус оплаты (`pending` -> `succeeded`/`canceled`) идемпотентно, логировать в `payments` и `event_log`.

### `GET /api/pricing/current`
Назначение: отдать текущую цену и время следующего переключения для таймера страницы `/pay`.

## Хранилище Google Sheets
- Лист `leads`: `lead_id`, `created_at`, `name`, `phone`, `consent_privacy`, `page`, `utm_*`, `first_touch_*`, `last_touch_*`.
- Лист `payments`: `payment_id`, `lead_id`, `status`, `amount`, `price_stage`, `created_at`, `updated_at`, `utm_*`.
- Лист `event_log`: `event_id`, `event_name`, `lead_id`, `payment_id`, `payload`, `created_at`.

## UTM-схема
- Поддерживаем параметры: `utm_source`, `utm_medium`, `utm_campaign`, `utm_content`, `utm_term`.
- Доп. параметры для теста: `utm_geo`, `utm_angle`, `ad_id`.
- first-touch и last-touch сохраняем в cookie + localStorage (TTL 30 дней).
- При переходах `/` -> `/pay` -> `/pay/success` сохраняем один `lead_id` и переносим UTM без потерь.

## YooKassa: обязательные требования
- `YOOKASSA_SHOP_ID` и `YOOKASSA_SECRET` только на сервере.
- Цена рассчитывается только на сервере по текущему МСК-времени.
- Диапазоны цены:
  - 2026-03-10 00:00 -> 2026-03-12 23:59: 1 990 ₽;
  - 2026-03-13 00:00 -> 2026-03-15 23:59: 2 490 ₽;
  - 2026-03-16 00:00 -> 2026-03-18 23:59: 2 990 ₽;
  - 2026-03-19 00:00 -> 2026-03-22 23:59: 3 490 ₽.
- Платеж создаем с `idempotence_key`.
- Финализация доступа только по webhook `succeeded`.
- Продавец в платежной части: самозанятый Семьянов Виталий Сергеевич.
- Статус перед запуском трафика: закрыть донастройку чеков/54-ФЗ.

## Mobile-first требования (обязательно)
- На `/` и `/pay` главный CTA виден без скролла.
- Sticky-кнопка CTA внизу экрана на мобильном.
- Высота интерактивных элементов не меньше 44px.
- Шрифт в `input` не меньше 16px.
- Корректный рендер на ширине 320px.
- Таймер на `/pay` не ломает верстку.

## Тесты и приемка
| Сценарий | Ожидаемый результат |
|---|---|
| Отправка формы без телефона | Успешно, лид сохраняется, редирект на `/prechat-instruction` |
| Отправка формы без согласия | Ошибка валидации, отправка блокируется |
| Передача UTM между страницами | UTM и `lead_id` сохраняются до `/pay/success` |
| Оплата успешна | webhook ставит `succeeded`, доступ в закрытый чат доступен |
| Оплата отменена | Возврат на `/pay?payment=failed`, оплата доступна повторно |
| Повтор webhook | Дубликаты в `payments` и событиях не создаются |
| Переключение цены в 00:00 МСК | Цена и таймер обновляются корректно |
| Mobile QA | CTA и форма кликабельны, блоки читаемы |

## Предрелизный чеклист
- ENV заполнены: `SHEET_ID`, `GOOGLE_SERVICE_ACCOUNT`, `TG_BOT_TOKEN`, `TG_CHAT_ID`, `YOOKASSA_SHOP_ID`, `YOOKASSA_SECRET`.
- Подставлены финальные Telegram-ссылки: чат предзаписи и закрытый чат.
- На всех страницах есть контакт `t.me/ira_deni` и ссылка `умныйживот.рф/privacy`.
- Юридический блок оплаты заполнен на продавца: самозанятый Семьянов Виталий Сергеевич.
- Пройден dry-run YooKassa в тестовом и боевом режиме.

## Зафиксированные допущения
- Google Sheets используется как основной учет лидов, оплат и событий.
- Доступ в закрытый чат выдается через `/pay/success` с ручным апрувом админа.
- Рекламный запуск не стартует до завершения настройки чеков/54-ФЗ.
- Отсутствие телефона не блокирует лид, если есть имя и согласие.

- На странице `/` используем более выгодный для клика текст CTA: `Попробовать бесплатно`.
- CTA у формы на `/`: `Вступить в чат и начать бесплатно`.
- На странице `/pay` единый CTA: `Оплатить доступ за {CURRENT_PRICE} ₽`.

### Логика переходов V3 (без изменений)
- Реклама -> `/` -> форма предзаписи -> `/prechat-instruction`.
- `/prechat-instruction` -> кнопка вступления в чат предзаписи.
- Из бесплатного чата -> переход на `/pay`.
- `/pay` -> YooKassa -> `/pay/success`.
- `/pay/success` -> кнопка вступления в закрытый чат.

### Контентные ограничения V3
- Не упоминаем на лендинге оффер `Свой темп`.
- Не показываем публично спеццену для теплой базы.
- Таймер только на странице `/pay`.
- В текущей версии оставляем примерные кейсы; по мере накопления фактуры обновляем на реальные без изменения структуры блока.
