## Правила работы

### Перед выполнением любой задачи:
1. Прочитай этот файл (AGENTS.md)
2. Прочитай brief.md нужного проекта — пойми контекст
3. Прочитай decisions.md — не противоречь принятым решениям
4. Проверь log.md — пойми, на каком мы этапе

### При создании или обновлении файлов:
1. Сохраняй результат в соответствующий .md файл проекта
2. Добавь запись в log.md проекта с датой и кратким описанием
3. Если нужно моё решение — отметь в log.md как **[РЕШЕНИЕ ТРЕБУЕТСЯ]**

### Формат:
- Все файлы — markdown
- Язык — русский
- Стиль — конкретный, без воды, с примерами
- Если не хватает информации — скажи, что именно нужно узнать

### Чего не делать:
- Не выдумывай факты обо мне или моих проектах
- Не меняй brief.md и decisions.md без моей команды
- Не удаляй существующий контент в файлах, только дополняй

## Мой стиль
- Простой язык, без корпоративного буллшита
- Конкретные действия, а не абстрактные советы
- Если нужна информация от меня — спроси, не додумывай
