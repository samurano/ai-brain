# D'Arky Dance — Dashboard Manual (VK Ads)

## Инструкция для руководителя

### 1) Как открыть таблицу
Откройте ссылку:
https://docs.google.com/spreadsheets/d/1fXa9j8P7qzv61II08C6HZJXr5Wmztgit1FmVAlqpL68/edit

Рабочий лист для вас: `Сводка`.

### 2) Как читать «Сводку»
- `Общий расход, ₽` — сколько реально потрачено на рекламу за выбранный период.
- `Количество заявок, шт` — сколько заявок пришло из рекламы.
- `Стоимость заявки (CPL), ₽` — средняя цена одной заявки.
- `Пришли на пробный, шт` — сколько людей реально дошли до пробного.
- `Стоимость пришедшего, ₽` — сколько стоит один дошедший до пробного.
- `Конверсия заявка → пробный` — доля заявок, которые дошли до пробного.
- `Купили абонемент, шт` — сколько людей купили абонемент.
- `Стоимость клиента (CAC), ₽` — сколько стоил один купивший клиент.
- `Неделя / Расход / Заявки / CPL` — динамика по неделям в выбранном периоде.
- `Месяц / Потрачено / Заявки / CPL` — месячный срез по всей накопленной истории.
- `WoW расход, WoW заявки, WoW CPL` — изменение последней недели к предыдущей.
- `Доля лидов из A06` — сколько лидов пришло из high-intent сегмента ключевых фраз.
- `Доля бюджета в сегментах с 0 лидами` — какая часть бюджета ушла в неэффективные adset без лидов.

Важно:
- Все суммы в ₽ на дашборде считаются **с НДС (+20%)**.
- Кабинет VK показывает суммы без НДС, а в дашборде вы видите управленческий gross.
- Ключевые рублёвые метрики в сводке округляются до целых значений.

### 3) Что означает светофор
- `🟢` — цена заявки в норме (ниже целевой).
- `🟡` — цена заявки выше цели, но в допустимом диапазоне.
- `🔴` — цена заявки слишком высокая, реклама работает неэффективно.

Пороговые значения:
- зелёный: `< 1320 ₽`
- жёлтый: `1320-1584 ₽`
- красный: `> 1584 ₽`

### 4) Когда звонить маркетологу
Если на «Сводке» `🔴` держится 3 дня подряд — сразу звоните/пишите маркетологу.

### 5) Как заполнять лист «Пробные → Абонементы»
Заполняйте по одной строке на человека:
- `дата`
- `имя`
- `источник (реклама/кампания)`
- `пришёл на пробный (да/нет)`
- `купил абонемент (да/нет)`
- `сумма`

Важно:
- В колонках `пришёл` и `купил` выбирайте только `да` или `нет` из выпадающего списка.

## Инструкция для маркетолога

### 1) Как экспортировать из VK Ads
Для каждого периода выгружайте 3 уровня:
1. `campaign`
2. `adset`
3. `ad`

Периоды:
- неделя (последние 7 дней)
- месяц (с 1-го числа по сегодня или закрытый месяц)

### 2) Какие поля важны в фактической raw-выгрузке VK
Минимум:
- `Дата`
- `ID/Название` сущности уровня
- `Показы`
- `Клики`
- `Потрачено всего, ₽`
- `Лиды`

Дополнительно (если есть):
- `CTR, %`
- `eCPC, ₽`
- `Цена за лида, ₽`
- `eCPM, ₽`
- `Охват на конец периода`
- `Частота на конец периода`
- `Статус`

### 3) Как назвать файл
Допустимы оба варианта:
- Рекомендуемый шаблон: `YYYY-MM-DD__YYYY-MM-DD__vkads__level-{campaign|adset|ad}__export.csv`
- Raw имя из VK (без переименования), например:
  - `ads_daily_report 01.01.2026 - 04.03.2026.xlsx`
  - `groups_daily_report 01.01.2026 - 04.03.2026.xlsx`
  - `banners_daily_report 01.01.2026 - 04.03.2026.xlsx`

### 4) Куда положить файл
В папку:
`darky-dance/marketing/vk-ads/data/`

### 4.1) Как называть кампании, группы и объявления (новый стандарт)
Чтобы ты и агент одинаково понимали, что внутри связки, используем простой формат:

1. Кампания:
`VK | <тип> | Парная бачата | <месяц>`

Примеры:
- `VK | Лиды | Парная бачата | Мар26`
- `VK | Ретаргет | Парная бачата | Мар26`

2. Группа (adset):
`[Axx] <сегмент> | <пол+возраст> | НСК`

Примеры:
- `[A06] Ключевые фразы | Ж 24-35 | НСК`
- `[A07] Конкуренты | М 27-38 | НСК`

3. Объявление (ad):
`[Cxx] <угол> | <формат> | <оффер> | vN`

Примеры:
- `[C01] Боль | Статика | Пробный 800 | v1`
- `[C03] Соцдоказ | Видео 15с | Пробный 800 | v2`

Правила:
- Всегда сохраняй `[Axx]` и `[Cxx]`.
- Один разделитель: только ` | `.
- Если меняешь гипотезу — поднимай версию `v2`, `v3`, а не переименовывай старое.

### 5) Какую команду дать Claude Code (точный текст)
```text
Загрузи raw-выгрузки VK Ads из darky-dance/marketing/vk-ads/data/ в Google Sheet "D'Arky Dance — Реклама ВК", обнови лист "Данные", пересчитай "Сводка" и "Кампании" с учётом НДС (+20% ко всем рублёвым метрикам), затем коротко отчитай итоги (строк, расход net/gross, заявки, CPL gross).
```

### 6) Что делает агент при команде «загрузи данные»
1. Читает все CSV/XLSX в `data/` (кроме `README.md`).
2. Определяет уровень отчёта (`campaign/adset/ad`) по имени файла, листу и колонкам.
3. Нормализует столбцы в каноническую схему листа `Данные`.
4. Делает `full refresh` листа `Данные` (без дублей).
5. Пересчитывает `Сводка` и `Кампании` в gross-метриках (с НДС).

### 7) Как интерпретировать доп. KPI на «Сводка»
- `WoW расход, %`:
  - рост = расширение инвестиций в трафик,
  - падение = сокращение бюджета/доставки.
- `WoW заявки, %`:
  - рост = улучшение объёма лидов,
  - падение = просадка потока.
- `WoW CPL, %`:
  - рост = ухудшение эффективности,
  - снижение = улучшение эффективности.
- `Доля лидов из A06`:
  - чем выше, тем больше лидов из high-intent семантики.
- `Доля бюджета в сегментах с 0 лидами`:
  - чем ниже, тем меньше потерь на неработающие adset.

### 8) Режим экономии tool calls (стандарт для всех следующих чатов)
- Цель на одну загрузку: `2-5` calls (максимум `10`).
- Базовый цикл:
  1. очистить `Данные` одним вызовом,
  2. записать весь массив одним batch-вызовом,
  3. опционально обновить `last_sync` одной ячейкой.
- В цикле загрузки не делать:
  - построчные append-операции,
  - поячейчные апдейты,
  - повторное форматирование и обновление графиков.
- Для отчёта по возможности считать метрики из локального файла, а не читать большие диапазоны из Sheets.

