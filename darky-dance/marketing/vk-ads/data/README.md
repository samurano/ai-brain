# VK Ads Data — как выгружать и загружать данные

## 1) Что выгружать из VK Ads
Для каждого отчётного периода выгружаем **3 уровня отчёта**:
1. Уровень `кампания` (`campaign`)
2. Уровень `группа объявлений` (`adset`)
3. Уровень `объявление` (`ad`)

Это нужно, чтобы в `analysis.md` видеть и общую картину, и узкие места по связкам.

---

## 2) Период выгрузки
- Еженедельный отчёт: последние 7 дней.
- Месячный отчёт: с 1 числа месяца по текущую дату (или полный закрытый месяц).

Рекомендация: по понедельникам выгружать сразу оба периода (неделя + месяц).

---

## 3) Какие столбцы нужны

### Минимально обязательные (без них загрузка не запускается)
- `Дата`
- ID и название сущности уровня (кампания / группа / объявление)
- `Показы`
- `Клики`
- `Потрачено всего, ₽` (или аналог расхода)
- `Лиды` (или аналог результата по лидам)

### Желательные (если есть — загружаем; если нет — не блокируем)
- `Охват`
- `Частота`
- `CPM`
- `CPL`
- `Плейсмент`
- `CTR`
- `CPC`
- `Статус`

Важно: в текущих raw-выгрузках VK поле `Плейсмент` обычно отсутствует. Это допустимо.

---

## 4) Формат файла
- `CSV UTF-8` и `XLSX` **равноправны**.
- Можно класть файлы в том формате, в котором выгрузил VK Ads, без ручной конвертации.

---

## 5) Именование файлов

### Рекомендуемый шаблон
`YYYY-MM-DD__YYYY-MM-DD__vkads__level-{campaign|adset|ad}__export.csv`

Примеры:
- `2026-03-01__2026-03-07__vkads__level-campaign__export.csv`
- `2026-03-01__2026-03-07__vkads__level-adset__export.csv`
- `2026-03-01__2026-03-07__vkads__level-ad__export.csv`

### Допустимо
Raw-имена из VK Ads **без переименования**, например:
- `ads_daily_report 01.01.2026 - 04.03.2026.xlsx`
- `groups_daily_report 01.01.2026 - 04.03.2026.xlsx`
- `banners_daily_report 01.01.2026 - 04.03.2026.xlsx`

Уровень отчёта в этом случае определяется агентом по имени файла и/или структуре колонок.

### Важно про нейминг внутри кабинета (campaign/adset/ad)
Для новых запусков используем короткий стандарт:
- Кампания: `VK | <тип> | Парная бачата | <месяц>`
- Группа: `[Axx] <сегмент> | <пол+возраст> | НСК`
- Объявление: `[Cxx] <угол> | <формат> | <оффер> | vN`

Это помогает восстанавливать срезы по аудиториям и креативам даже в raw-выгрузке без явных полей.

---

## 6) Куда положить файл в репозитории
Все выгрузки класть сюда:
`darky-dance/marketing/vk-ads/data/`

Не класть выгрузки в другие папки, чтобы не ломать цикл анализа.

---

## 7) НДС и суммы в ₽
- В кабинете VK Ads суммы обычно указаны **без НДС (net)**.
- В дашборде для управленческого учёта все денежные метрики считаются как **gross**:
  - `gross = net * 1.2`
- Это касается расхода, CPC, CPL, CAC и других рублёвых метрик.

---

## 8) Что написать агенту после загрузки
После загрузки файлов отправь агенту сообщение по шаблону:

```text
Загрузил VK Ads выгрузки за период [дата_начала — дата_конца] в darky-dance/marketing/vk-ads/data/.
Файлы:
- [имя файла 1]
- [имя файла 2]
- [имя файла 3]

Загрузи данные в таблицу, пересчитай метрики и проверь сводку.
После этого обнови:
1) darky-dance/marketing/vk-ads/analysis.md
2) darky-dance/marketing/vk-ads/recommendations.md
3) darky-dance/marketing/vk-ads/tests-log.md
```

---

## 9) Быстрый чек перед отправкой
- [ ] Есть 3 файла (campaign/adset/ad по сути)
- [ ] Формат CSV или XLSX
- [ ] В каждом файле есть дата, ID/название уровня, показы, клики, расход, лиды
- [ ] Файлы лежат в `darky-dance/marketing/vk-ads/data/`

---

## 10) Экономный режим загрузки (чтобы не сжигать лимит Composio)
- Цель: `2-5` tool calls на один цикл обновления.
- Допустимый максимум: `<=10` calls.

Рекомендуемый порядок:
1. Агент локально парсит файлы в `data/` (0 calls к Sheets).
2. Одним вызовом очищает диапазон данных листа `Данные`.
3. Одним batch-вызовом записывает весь массив в `Данные`.
4. Опционально одним вызовом обновляет `last_sync`.

Что не делать:
- append в цикле по строкам,
- поячейчные изменения,
- регулярное форматирование/перекраску/перестройку графиков при каждой загрузке,
- лишние чтения больших диапазонов из Sheets.
